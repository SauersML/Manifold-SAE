conventions. Generated by the integration audit (`manifold_sae/integration.py`).

## Composition observations

- **Forward signature drift**: `crm` and `crosscoder` take a *list of tensors* (multi-layer);
  every other variant takes a single `(B, D)` tensor. The integration layer detects this and
  replicates `x` across layers when needed.
- **Loss-key drift**: `wasserstein` returns `total` not `loss`; `equivariant` and `Crosscoder`
  have no `.loss()` method (compute total in `forward()` dict or via standalone helpers);
  `adaptive_k` confusingly uses `recon` for the MSE scalar and `recon_out` for the recon tensor.
- **Recon-tensor location**: `recon_out` (adaptive_k), `recon` (wasserstein, equivariant,
  crm-as-`out.recons[0]`, crosscoder-as-`recons[0]`), `reconstruction` (ManifoldSAE attr).
- **dtype/device**: every variant accepts `.to(device)`. ManifoldSAE (gamfit) is pinned to its
  config dtype (float64 by default for the REML solve) and *rejects* mismatched input rather than
  silently promoting; the integration layer (`_call_loss`) casts X to the model's dtype at the
  boundary.
- **Save/load**: every variant supports `torch.save(model.state_dict())` round-trip; only
  ManifoldSAE adds a schema-version byte (`_SAE_SCHEMA_VERSION`).
